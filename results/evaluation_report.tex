\subsection{RAG Pipeline}

The system implements a four-stage retrieval pipeline:

\begin{enumerate}[leftmargin=*]
    \item \textbf{Query Embedding:} Transform input query into 768-dimensional dense vector using MPNet
    \item \textbf{Vector Retrieval:} Approximate nearest neighbor search in Qdrant vector store
    \item \textbf{Score Filtering:} Apply relevance threshold (0.5) to filter low-confidence matches
    \item \textbf{Result Ranking:} Return top-K results sorted by cosine similarity
\end{enumerate}

\subsection{Evaluation Metrics}

\subsubsection{Retrieval Quality Metrics}

\begin{itemize}[leftmargin=*]
    \item \textbf{Recall@K:} Proportion of queries where at least one relevant document appears in top-K results
    \begin{equation}
        \text{Recall@K} = \frac{|\{q : \exists d \in \text{top-K}(q), \text{sim}(q,d) \geq \theta\}|}{|Q|}
    \end{equation}
    
    \item \textbf{Top-K Accuracy:} Equivalent to Recall@K for single-relevant-document scenarios
    
    \item \textbf{Mean Reciprocal Rank (MRR):} Average of reciprocal ranks of first relevant result
    \begin{equation}
        \text{MRR} = \frac{1}{|Q|} \sum_{i=1}^{|Q|} \frac{1}{\text{rank}_i}
    \end{equation}
    
    \item \textbf{nDCG@K:} Normalized Discounted Cumulative Gain measuring ranking quality
    \begin{equation}
        \text{nDCG@K} = \frac{\text{DCG@K}}{\text{IDCG@K}} = \frac{\sum_{i=1}^{K} \frac{2^{rel_i} - 1}{\log_2(i+1)}}{\text{IDCG@K}}
    \end{equation}
\end{itemize}

\subsubsection{Robustness Metrics}

\begin{itemize}[leftmargin=*]
    \item \textbf{Paraphrase Robustness:} Consistency of results across semantically equivalent queries
    \item \textbf{False Positive Rate:} Rate of high-confidence results for out-of-scope queries
    \item \textbf{Hallucination Rate:} Rate of confidently wrong retrievals
\end{itemize}

\subsubsection{Performance Metrics}

\begin{itemize}[leftmargin=*]
    \item \textbf{Stage-wise Latency:} Time breakdown for embedding, retrieval, and re-ranking
    \item \textbf{P50/P95/P99 Latency:} Percentile latency distribution
    \item \textbf{Cache Performance:} Hit rate and latency improvement from caching
\end{itemize}

\subsection{Relevance Threshold}

A result is considered \textbf{relevant} if its cosine similarity score $\geq 0.5$. This threshold was calibrated to balance precision and recall for real estate domain queries.

Section F tested 15 paraphrased versions of existing queries. The system correctly retrieved relevant results for 11 out of 15 queries (73.3\%).

\textbf{Failure Analysis:} Paraphrase failures typically occurred when:
\begin{itemize}
    \item Query rephrasing used synonyms not present in the document corpus
    \item Structural changes altered the semantic emphasis
    \item Domain-specific terminology was replaced with colloquial expressions
\end{itemize}

\subsection{Optimization Recommendations}

To achieve the P95 $<$ 2000ms target:
\begin{enumerate}
    \item \textbf{GPU Acceleration:} Move embedding inference to GPU (estimated 5-10x speedup)
    \item \textbf{Batch Processing:} Pre-compute embeddings for common query patterns
    \item \textbf{Index Optimization:} Tune Qdrant HNSW parameters (ef, m values)
    \item \textbf{Model Quantization:} Use INT8 quantized embedding model
\end{enumerate}
